History of VCS

At the beginning version control systems (VCS) were centralized. One server stored the true copy of a project. But they had one major flaw: if the server broke, nobody could access their projects. Therefore a new version control system arrived. It was Distributed version control systems didn't have such flaw and are widely used nowadays. Most popular is github with over 100 million developers.


Mercurial and Github were competing in being the one VCS for everyone. Mercurial was straight-forward and Github was more complex. Surprisingly, the more complex option won and Mercurials is currently used only by few projects.

List of historical moments

  • 1972 - SCCS (Bell Labs)
  • 1982 - RCS (Purdue Univ.)
  • 1986 - CVS (Disc Grune)
  • 2000 - Subversion (CollabNet)
  • 2005 - Git (Linus Toryalds)
  • 2005 - Mercurial (Matt Mackall)
  • 2008 - Github Launched
  • 2017 - GitOps Term coined
  • 2023 - 100M Devs on Github
